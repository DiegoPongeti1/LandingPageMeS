╔══════════════════════════════════════════════════════════════════════════════╗
║  1. VISÃO GERAL DO PROJETO                                                 ║
╚══════════════════════════════════════════════════════════════════════════════╝

Este projeto é uma Landing Page da empresa "Micro&Soft", uma empresa de 
tecnologia localizada em Franca - SP. A página foi construída com Next.js 
usando o App Router, React com TypeScript e estilização via TailwindCSS.

O objetivo principal da página é apresentar a empresa, seus diferenciais,
empresas parceiras e informações de contato (telefone, e-mail, endereço 
e WhatsApp).

────────────────────────────────────────────────────────────────────────────────
  4.1  layout.tsx  —  Layout Raiz
────────────────────────────────────────────────────────────────────────────────
  Caminho: app/layout.tsx
  Tipo: Server Component

  O QUE FAZ:
    • Define a estrutura HTML raiz da aplicação (<html> e <body>).
    • Configura as fontes do Google: "Geist" (sans-serif) e "Geist_Mono" (mono).
    • Define os metadados da página:
        - Título: "Micro&Soft"
        - Descrição: "Empresa de tecnologia"
    • Aplica as variáveis de fonte e a classe "antialiased" ao <body>.
    • Renderiza os componentes filhos ({children}) dentro do body.

  DEPENDÊNCIAS:
    • next/font/google (Geist, Geist_Mono)
    • globals.css

────────────────────────────────────────────────────────────────────────────────
  4.2  page.tsx  —  Página Principal (Home)
────────────────────────────────────────────────────────────────────────────────
  Caminho: app/page.tsx
  Tipo: Server Component

  O QUE FAZ:
    • É a página principal da aplicação (rota "/").
    • Monta a estrutura da Landing Page usando um layout flexbox vertical.
    • Fundo branco (#ffffff), altura mínima da tela inteira (min-h-screen).
    • Ordem dos componentes renderizados:
        1. <Header />         → Cabeçalho com logo
        2. <CardsChamativos /> → Cards com diferenciais
        3. <Parceiros />      → Carrossel de parceiros
        4. <Infos />          → Informações de contato
        5. <Footer />         → Rodapé
    • O <main> usa "grow" para ocupar o espaço disponível.

  DEPENDÊNCIAS:
    • Header, Footer, Parceiros, Infos, CardsChamativos (de ./_components/)

  4.3  Header.tsx  —  Cabeçalho
────────────────────────────────────────────────────────────────────────────────
  Caminho: app/_components/Header.tsx
  Tipo: Server Component

  O QUE FAZ:

────────────────────────────────────────────────────────────────────────────────
  4.5  Parceiros.tsx  —  Carrossel de Parceiros
────────────────────────────────────────────────────────────────────────────────
  Caminho: app/_components/Parceiros.tsx
  Tipo: Client Component ('use client')

  O QUE FAZ:
    • Exibe a seção "Empresas parceiras da Micro&Soft".
    • Define um array de empresas parceiras com os campos:
        - nome, tipo, img, descricao, link (opcional)
    • Empresas cadastradas:
        1. Veromoc — Calçados (com link para site)
        2. Coberchapas — Chapas (com link para Instagram)
        3. Classe Couro — Couros (com link para site)
        4. Empresa 4 — Varejo (sem link)
        5. Empresa 5 — Varejo (sem link)
    • Cria um carrossel infinito duplicando o array 3x e usando CSS
      animation (translateX de 0 a -33.33%) com duração de 25 segundos.
    • Cada logo é um <button> clicável que, ao ser pressionado:
        - Salva a empresa selecionada no state (useState)
        - Abre o modal (setModalAberto(true))
        - Pausa a animação do carrossel (classe "pause-scroll")
    • Renderiza o componente <InfosEmpresas /> passando os dados da
      empresa selecionada e a função de fechar o modal.

  ESTADOS (useState):
    • empresaSelecionada (Empresa | null) — empresa clicada
    • modalAberto (boolean) — controla visibilidade do modal

  TYPE:
    • Empresa { nome, tipo, img, descricao, link? }

  DEPENDÊNCIAS:
    • next/image, react (useState), InfosEmpresas

────────────────────────────────────────────────────────────────────────────────
  4.6  InfosEmpresas.tsx  —  Modal de Detalhes da Empresa
────────────────────────────────────────────────────────────────────────────────
  Caminho: app/_components/InfosEmpresas.tsx
  Tipo: Server Component (sem 'use client', mas recebe props)

  O QUE FAZ:
    • Componente de modal que exibe detalhes de uma empresa parceira.
    • Recebe via props:
        - aberta (boolean) — se o modal está visível
        - empresa (Empresa | null) — dados da empresa
        - onFechar (função) — callback para fechar o modal
    • Se "aberta" for false ou "empresa" for null, retorna null (não renderiza).
    • Exibe:
        - Overlay escuro semitransparente (bg-black/50) cobrindo a tela toda
        - Card centralizado com:
            • Botão "×" para fechar (canto superior direito)
            • Nome da empresa (h3, bold)
            • Tipo da empresa (texto cinza)
            • Descrição da empresa
            • Botão "Saiba Mais" (se houver link) — vermelho escuro (#800000),
              que abre o link em nova aba

  TYPES:
    • Empresa { nome, tipo, img, descricao, link? }
    • InfosEmpresasProps { aberta, empresa?, onFechar }

  DEPENDÊNCIAS:
    • React

────────────────────────────────────────────────────────────────────────────────
  5.1  button.tsx  —  Componente de Botão (shadcn/ui)
────────────────────────────────────────────────────────────────────────────────
  Caminho: components/ui/button.tsx

  O QUE FAZ:
    • Componente de botão reutilizável baseado no padrão shadcn/ui.
    • Usa class-variance-authority (cva) para gerenciar variantes de estilo.
    • Variantes disponíveis:
        VARIANT         | ESTILO
        default         | Fundo primário, texto claro, sombra
        destructive     | Fundo vermelho, texto claro
        outline         | Borda, fundo transparente
        secondary       | Fundo secundário
        ghost           | Sem fundo, hover com destaque
        link            | Estilo de link com underline

    • Tamanhos disponíveis:
        SIZE    | DIMENSÕES
        default | h-9, px-4, py-2
        sm      | h-8, px-3, texto xs
        lg      | h-10, px-8
        icon    | h-9, w-9

    • Suporta a prop "asChild" via @radix-ui/react-slot para
      renderizar como outro elemento.

  DEPENDÊNCIAS:
    • @radix-ui/react-slot
    • class-variance-authority (cva)
    • lib/utils (cn)
